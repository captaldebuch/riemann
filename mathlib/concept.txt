## Tool 1 — Half-line Mellin–Plancherel

This is the most natural one to build first.

The standard trick is the logarithmic change of variables:

```text id="od2wu8"
x = exp(t)
```

Define the unitary map morally by:

```text id="k9x80e"
(Uf)(t) = exp(t/2) · f(exp t)
```

Then the Mellin transform on the critical line becomes a Fourier transform:

```text id="c2qhcv"
M f(1/2 + iτ) = Fourier(Uf)(τ)
```

up to normalization conventions.

So the Mathlib tool should not start as “prove Nyman–Beurling.” It should start as:

```lean id="bm0a4m"
def logPullbackL2 : L2((0,∞), dx) ≃ₗᵢ[ℂ] L2(ℝ, dt)
```

Then prove:

```lean id="zyu7ej"
theorem mellin_eq_fourier_logPullback :
  Mellin f (1/2 + I*t) = Fourier (logPullbackL2 f) t
```

Then:

```lean id="gk6ieb"
theorem mellin_plancherel_halfline :
  ∫ t, ‖Mellin f (1/2 + I*t)‖^2 = C * ‖f‖^2
```

This is a clean Mathlib contribution because it is general, reusable, and independent of RH.

## Tool 2 — Mellin evaluation continuity / zero detection

This is more project-specific but still can be generalized.

You need a theorem of the following form:

```text id="z6u5as"
For s with Re(s)>1/2, Mellin evaluation at s is continuous
on the relevant Nyman–Beurling closure/domain.
```

Then the proof becomes:

```text id="dcy2ce"
If ζ(s₀)=0:
  every generator has Mellin value 0 at s₀.
By closure:
  χ also has Mellin value 0 at s₀.
But Mellin(χ)(s₀)=1/s₀ ≠ 0.
Contradiction.
```

The formal tool should probably not be called “zero-detection” at first. It should be stated as a general continuity lemma:

```lean id="q826q4"
theorem continuous_mellin_eval_on_halfplane
    (s : ℂ) (hs : 1/2 < s.re) :
    ContinuousLinearMap ... ...
```

Then the RH-specific zero-detection theorem is a downstream application.
